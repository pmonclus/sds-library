typedef void(* SdsConfigCallback) (const char *table_type, void *user_data)

Callback for config updates (device role only).

Called when a device receives new configuration from the owner. The config data has already been deserialized into the table struct when this callback is invoked.

Parameters
table_typeName of the table that received the update
user_dataUser-provided context from sds_on_config_update()
See also
sds_on_config_update

typedef void(* SdsDeviceEvictedCallback) (const char *table_type, const char *node_id, void *user_data)

Callback for device eviction (owner role only).

Called when a device is evicted from the status table after the grace period expires following an LWT (device offline) message.

Parameters

◆ SdsVersionMismatchCallback

typedef bool(* SdsVersionMismatchCallback) (const char *table_type, const char *device_node_id, const char *local_version, const char *remote_version)

Callback for schema version mismatch detection (owner role only).

Called when an owner receives a message from a device running a different schema version. The callback can decide whether to accept or reject the message.

Parameters
table_typeThe table type where the mismatch was detected
device_node_idThe node ID of the device with different version
local_versionOur schema version string
remote_versionThe device's schema version string
Returns
true to accept the message anyway, false to reject it
See also
sds_on_version_mismatch, sds_get_schema_version
